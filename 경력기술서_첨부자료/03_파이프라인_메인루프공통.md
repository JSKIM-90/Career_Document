# 0-produce auto SKILL 공통 — 메인 루프 보일러플레이트

> 이 문서는 `produce-standard-auto` / `produce-advanced-auto` / `produce-3d-standard-auto` / `produce-3d-advanced-auto` 4개 SKILL의 메인 루프(Phase 2 / Phase 3 / 종료 보고 / 금지 사항)가 공통으로 따르는 절차다. 변형별 SKILL.md는 변형 고유의 검증/큐 업데이트 등 추가 단계만 inline으로 명시한다.
>
> **변형별 SKILL.md는 본 문서를 §X 형태로 참조한다 (예: "§Phase 2 공통 절차 + 추가: ADVANCED_QUEUE.md 상태 업데이트").**

---

## §Phase 2 공통 절차 — Agent 반환 후

모든 변형이 다음 단계를 수행한다.

### 1. 생성된 파일 존재 확인

변형별 SKILL.md가 정의한 산출물 경로를 `ls`로 확인한다 (예: `register.js`, `beforeDestroy.js`, `CLAUDE.md`, view/style/preview).

### 2. manifest.json JSON 유효성 검증

```bash
node -e "JSON.parse(require('fs').readFileSync('RNBT_architecture/DesignComponentSystem/manifest.json','utf8')); console.log('OK')"
```

### 3. 커밋

커밋 메시지 패턴 (변형별 SKILL.md가 `{대상 경로}` 부분을 변경할 수 있음):

```
feat: {대상 경로} 컴포넌트 자동 생산 — {Agent 요약 첫 줄}
```

### 4. 공통 실패 감지 — 즉시 중단

- 파일 누락 → "Phase 2 파일 검증 실패: {누락 목록}" 보고
- JSON 오류 → "manifest.json 파손: {오류}" 보고
- Agent가 실패를 보고 → Agent 요약 그대로 사용자에게 전달

> 변형별 SKILL.md는 추가 검증/실패 사유를 정의한다. 예:
> - Advanced: ADVANCED_QUEUE.md 상태 `대기` → `완료` 업데이트, 실패 시 `대기` 유지하여 재실행 가능.
> - 3D Advanced: 컴포넌트 루트 CLAUDE.md 세트 현황 표 갱신 확인 (3중 등록).

## §Phase 3 공통 — 다음 사이클

남은 대상이 있으면 Phase 0부터 다시 실행. 없으면 종료.

> 3D Advanced의 Standard 선행 필터로 건너뛴 항목은 그대로 "대기" 유지 — 사용자가 `produce-3d-standard-auto`로 Standard를 채운 뒤 본 SKILL을 재실행하면 자동 처리.

## §종료 보고 template

### 성공

```
✅ {차원} {단계} 전체 생산 완료.
생산된 {단위}: N개
커밋: N개
```

### 중단

```
⚠️ 중단: {사유}
현재까지 생산: N개
마지막 대상: {경로}
재개하려면 `/{SKILL 이름}`을 다시 실행하세요.
```

### 치환 예

| 변수 | Standard auto | Advanced auto |
|------|---------------|---------------|
| `{차원}` | `2D` 또는 `3D` | `2D` 또는 `3D` |
| `{단계}` | `Standard` | `Advanced` |
| `{단위}` | `컴포넌트` (2D) / `장비` (3D) | `변형` |
| `{SKILL 이름}` | `produce-standard-auto` 등 | `produce-advanced-auto` 등 |

> Advanced 변형은 중단 보고에 `큐 상태: "대기" 유지 (재실행 가능)` 1줄을 추가한다.
> 3D Advanced는 종료 보고에 `건너뛴 항목 (Standard 미생산): M개` 1줄을 추가한다.

## §공통 금지 사항

다음 3개는 모든 produce-*-auto SKILL에 동일하게 적용된다.

- ❌ 사용자에게 중간 승인 요청 (완전 자동 모드)
- ❌ Agent 호출 없이 메인이 직접 컴포넌트/변형 생산
- ❌ 실패를 덮고 다음 사이클 진행

> 변형별 SKILL.md는 추가 금지 사항을 정의한다. 예:
> - 모든 변형: 한 사이클 안에서 여러 컴포넌트/변형 생산 (Agent는 반드시 1 단위만)
> - 모든 변형: 신규 Mixin 생성 (반복 패턴은 반환 메모로만)
> - Advanced: Agent가 ADVANCED_QUEUE.md 직접 수정
> - 3D Standard auto: Advanced 변형 생산 (이 루프는 Standard 전용)
> - 3D Advanced auto: Standard 선행 필터 우회
