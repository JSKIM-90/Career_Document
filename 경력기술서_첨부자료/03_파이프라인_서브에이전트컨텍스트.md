## §공통 필수 제약

다음 5개는 모든 produce-*-auto Sub-agent 프롬프트에 동일하게 포함된다.

- **PostToolUse hook이 차단(stderr 출력)하면 그 출력에 따라 정정한 뒤 같은 도구를 재호출한다.** 1회 차단은 사이클 실패가 아니다 — 정정 후 통과하면 정상 진행. 정정 시도 후에도 반복 차단되면 그때 반환의 "발견한 문제"로 보고한다.
- **manifest.json 수정 후 다음 명령으로 JSON 유효성을 직접 검증한다:**
  ```bash
  node -e "JSON.parse(require('fs').readFileSync('RNBT_architecture/DesignComponentSystem/manifest.json','utf8')); console.log('OK')"
  ```
- **기존 완성된 컴포넌트 파일은 수정하지 않는다.** (단, 일부 변형은 컴포넌트 루트 CLAUDE.md의 세트 현황 표 갱신 등 명시적 예외가 있을 수 있다 — 변형별 SKILL.md에서 별도 지정.)
- **커밋은 하지 않는다.** 메인 루프가 커밋한다.
- **신규 Mixin 생성 금지.** 기존 Mixin + 커스텀 메서드 조합으로만 해결한다. 반복 패턴이 의심되거나 큐 설명에 "신규 Mixin: XxxMixin"이라고 명시되어 있어도, 이 사이클에서는 커스텀 메서드로 완결하고 반환의 "발견한 문제/의문점"에 `Mixin 승격 후보: {설명}`로 기록만 남긴다. `create-mixin-spec` / `implement-mixin` 스킬은 호출하지 않는다.

> 변형별 SKILL.md가 추가 제약을 정의할 수 있다. 예:
> - 2D 공통: MD3 명세 WebFetch → 실패 시 WebSearch로 대체. CSS는 `#{component}-container { ... nesting ... }` 형태.
> - 2D Standard/Advanced 공통: 컴포넌트 CLAUDE.md의 cssSelectors 계약을 HTML과 JS에서 일치.
> - Advanced 공통: ADVANCED_QUEUE.md는 수정하지 않는다 (메인 루프가 상태 업데이트).
